Übernehmen/Verwerfen

Das XML-Speichermodul bildet den Kern von lyra.



Wichtig!
Es wird zwischen dem permanenten Zustand (auf dem Speichermedium, HD) und dem flüchtigen Zustand (workspace = Programm-Speicherbereich) unterschieden.
Die Daten werden beim Programmstart von der XML-Datei in den workspace geladen (load). Zu Beginn sind also die gespeicherten Daten und die Daten im workspace identisch. Wenn Sie Änderungen vornehmen (neu erstellen, editieren, löschen), erfolgen diese nur im workspace. Sie werden also nicht direkt abgespeichert, sondern werden erst in die XML-Datei geschrieben, wenn Sie die Änderungen übernehmen (Commit) ("Datei"->"Übernehmen!" oder Ctrl+S). Wenn das Übernehmen funktioniert, erscheint am unteren Fensterrand die Statusmeldung "Änderungen erfolgreich übernommen...". Wenn an dieser Stelle "Fehler beim Übernehmen" steht und es nach wiederholtem Versuch nicht funktioniert, müssen Sie damit rechnen, dass Änderungen verloren gehen.
Beim Beenden von lyra werden Sie benachrichtigt, falls der workspace sich von den gespeicherten Daten unterscheidet.

Über "Datei"->"Änderungen verwerfen!"->"Ok" können Sie den workspace auf die zuletzt übernommene Version zurücksetzten (Rollback)


Backups:

Es wird kein log über Änderungen geführt! D.h. sie können einen übernommenen Zustand nicht zurücksetzen.

Ein Backup können Sie aber natürlich selber manuell erstellen. Exportieren Sie einfach in regelmässigen Abständen den Datenbankinhalt und speichern Sie die XML-Datei an einem sicheren Ort. Damit sind Sie in der Lage, frühere Datensätze wieder zu importieren.
(Sie können von der XML-Datei {lyra}\data\curtext.xml auch direkt eine Kopie erstellen und diese dann einfach wieder erstetzen, wenn Sie die Daten wiederherstellen wollen. Sie sollten aber diese Datei sonst unbedingt unverändert lassen.)
